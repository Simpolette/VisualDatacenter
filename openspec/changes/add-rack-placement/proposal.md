## Why

Currently, adding a rack to a datacenter room is not supported in the 3D viewport UI. Enabling users to visually place racks directly onto the room's floor grid via an interactive click-and-drag gesture makes room layout design intuitive, realistic, and CAD-like.

## What Changes

- **Add Rack Mode (Placement State)**: Introduce a placement state in the room detail view, toggled by the "Add Rack" button. The top toolbar will display placement instructions and a "Cancel" action.
- **Click-and-Drag Placement Interaction**: In placement mode, allow the user to click down on a grid cell to specify the rack's center coordinate (`posX`, `posY`) and drag away to orient the rack's front-facing direction (snapping rotation to 90-degree increments). Releasing the mouse locks the ghost rack position.
- **Sidebar Reuse & Extraction**: Refactor the slide-out sidebar layout into a reusable `RightSidebar` container inside `frontend/src/components/Sidebar/` to handle both inspecting rack details and inputting new rack configuration.
- **New Rack Creation Form**: Provide a custom configuration form inside the reusable sidebar when in placement mode, enabling the user to edit the rack name (autogenerated defaults), specify total Units (42U or 44U), and commit creation via a "Create Rack" action.
- **Dynamic Store Integration**: Hook up store actions to call backend `POST /api/v1/rooms/{roomId}/racks` and automatically insert the newly created rack into the active 3D canvas viewport without page reload.
